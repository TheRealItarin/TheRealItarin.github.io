Attended conferences and tech&Meets

The 5 level of infrastructure as code

I was looking forward to attending this conference, because I just had hand on experience with this topic on my internship place. While it was an interesting experience, sadly they didn't talk about the generic concept too much. It was more like "This is how we implemented infrastructure as code in our company" rather than "This is what infrastructure as code is". They provided only examples for their company's ecosystem, not even a glance to other platforms. While Azure and Microsoft infrastructure is very extensive, I felt that the conference could have provided more insight into the broader applications and benefits of infrastructure as code. To summarize this tech and meet, it was a nice experience but the whole presentation felt like they are not talking about the topic they stated, and they were here only to headhunt future employees for their company.

Projects

Contributing to the European data strategy project.

As part of my exchange program at Fontys, I have contributed to the European data strategy project, with a team of four. Our work included the Common European data spaces initiative, where we were tasked to find and research tools which provide data sovereignty in data spaces environment. To fulfill this task, we conducted extensive research, analysis, and comparison for multiple solutions. Also to be able to properly test the tools we create our own data space testing environment.

Research on DNS attack vectors and trends

As one of my projects, I conducted a preliminary research on the most used cyber security attack vectors in 2025. After analyzing the data, I identified two main trends. DNS flood and amplification attacks. Because of the increasing reliance on DNS for various services, these attacks pose a significant threat to network security. So after completing this preliminary research, I started my project to understand how these attacks works and to find ways to mitigate these threats.

Creating a SaaS (retro shell)

I'm a huge fan of retro games, so when I get free hands for my Network engineering project, I decided to create a SaaS for Dos retro games. Naturally, I had to include cybersecurity measures to ensure the platform was secure. I gained in depth knowledge about TLS/SSL protocols, Diffie-Hellman key exchange, and other cryptographic concepts. It was a challenging but rewarding experience, both professionally and personally. The project can be found on my GitHub: https://github.com/TheRealItarin/Specialization-project-Fontys/tree/main

Web authentication project

This project focused on implementing secure web authentication mechanisms, ensuring user data protection and privacy. I gained valuable experience in handling user sessions, and securing API endpoints. The project uses Vue as frontend and Laravel as backend. The main goal of this project was to research various authentication methods and implement the one which we found the ost fitting in the context of security. The project can be found on my GitHub: https://github.com/TheRealItarin/AuthenticationBoK

Honeypot project

Me and my friend worked on a honeypot project, which was a web system to detect and log attacker behavior. Built on LEMP stack with Elasticsearch, Filebeat, Kibana for analytics. Implemented ModSecurity to detect and block malicious HTTP traffic. Automated log collection and anomaly detection.
